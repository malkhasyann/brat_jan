1) արտհայտություններ
    Բռատ ջան, արտահայտությունները կարող են լինել բարդ, այսինքն կարող են
պարունակել բարդ թվաբանություն կամ փոփոխականների անուններ։
Թվաբանական գործողությունները ունեն առաջնահերթություններ,
օրինակ`
        ( 26 + 4 ) * 2
արտահայտության արժեքը 60 է։ Ինչպես նկատեցիք, արտահայտության բոլոր
էլեմենտները բաժանված են բացատներով, անգամ փակագծերը։ Որպեսզի արտահայտությունը
աշխատի, պետք է բոլոր էլեմենտները բաժանված լինեն բացատներով։

2) փոփոխականներ
    Բռատ ջան, փոփոխականները հայտարարվում են հետևյալ կերպ.
        bratjan name = value
որտեղ bratjan-ը նշանակում է որ փոփոխականը հայտարարվում է
առաջին անգամ, name-ը փոփոխականի անունն է, իսկ value-ն՝ արժեքը։
Փոփոխականների անունները պետք է սկսվեն լատիներեն տառով և
կարող են պարունակել միայն լատիներեն տառեր կամ թվեր։
    Եթե ցանկանում եք արդեն հայտարարված փոփոխականին վերագրել
նոր արժեք, ապա ընդամենը բաց եք թողում bratjan֊ը, օրինակ`
        name = new_value
Փոփոխականները լինում են 3 տիպի՝ տող, թիվ, բուլյան։

3) bratwrite
    Բռատ ջան, լեզուն ունի էկրանին ինֆորմացիա արտածելու հնարավորություն
bratwrite ֆունկցիայի միջոցով, օրինակ
        bratwrite( 25 + 25 )
        bratwrite('') -> դատարկ տող
Ուշադրություն դարձրեք, արտածվող արտահայտությունը բացատներով առանձնացված է
ֆունկցիայի փակագծերից, հակառակ դեպքում կոդը չի աշխատի։

4) if-else
    Բռատ ջան, լեզուն ընդգրկում է ճյուղավորման
հրաման(կիսատ֊պռատ), այն ունի հետևյալ տեսքը.
            if condition
            {
                instructions
            }
            else
            {
                instructions
            }
Ցանկության դեպքում, կարող եք բաց թողնել else հրամանը։

5) Ցիկլեր
    Բռատ ջան, լեզուն ընդգրկում է նաև(նորից կիսատ֊պռատ) ցիկլային հրաման,
որն ունի հետևյալ տեսքը.
            while condition
            {
                instructions
            }

6) Լեզուն անգամ ունի namespace-երի stack, սակայն ցավն այն է,
որ դեռևս ամբողջությամբ դրա առավելություններից օգտվել հնարավոր չէ,
քանի որ չկա կոդի մարմինների ներդրման մեխանիզ, այսինքն հնարավոր չէ
if-else կամ while որևէ հրաման ներդնել մեկ այլ այդպիսի հրամանի մեջ։
    Նաև if-else կամ while հրամանների մարմինների մեջ խուսափեք դատարկ տողերից,
հակառակ դեպքում կոդը չի աշխատի։

Փոխարենը ունի սխալների սեփական նամակները և տողը #-ով քոմենթելու հնարավորություն։
